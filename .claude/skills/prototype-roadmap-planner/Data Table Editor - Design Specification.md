Data Table Editor - Design Specification

  Overview

  A spreadsheet-style editor plugin for managing all game data (items, abilities, enemies, NPCs, etc.) in a central, visual interface. Think "Google Sheets meets Godot Inspector" - familiar spreadsheet UX with game-specific validation and export.

  ---
  Core Concepts

  Tables

  A Table is a collection of rows with a defined schema (columns). Examples:
  - weapons - All weapon items
  - abilities_fighter - Fighter class abilities
  - enemies - Enemy stat blocks
  - loot_tables - Drop rate configurations

  Technical Implementation Notes

  UI Components Needed

  Godot Built-in:
  - Tree - For table sidebar (collapsible categories)
  - ItemList - Alternative for flat table list
  - LineEdit - Text input fields
  - SpinBox - Number inputs
  - OptionButton - Dropdowns
  - CheckBox - Boolean fields
  - TextEdit - Multiline text
  - FileDialog - Resource browser
  - HSplitContainer / VSplitContainer - Panel layout

  Custom Components to Build:
  - DataGrid - Spreadsheet-like grid (most complex)
  - SchemaField - Renders correct input for data type
  - ValidationBadge - Shows âœ“/âœ— with tooltip
  - FilterBar - Search + filter + sort controls
  - DiceInput - Specialized input for dice notation

  DataGrid Implementation Approach

  Option A: Custom Drawing (Complex)
  - Full control over rendering
  - Best performance for large tables
  - Most work to implement

  Option B: GridContainer with Dynamic Cells (Simpler)
  - Use Godot's built-in containers
  - Easier to implement
  - May have performance limits with 1000+ rows

  Option C: Tree with Columns (Recommended)
  - Tree node supports columns
  - Built-in selection, scrolling
  - Good enough for hundreds of rows
  - Fastest to implement

  Recommendation: Start with Option C (Tree), upgrade to custom if needed.

  Integration with Game Runtime

  Loading Data in Game

  # autoload/game_data.gd
  extends Node

  var weapons: Dictionary = {}
  var abilities: Dictionary = {}
  var enemies: Dictionary = {}

  func _ready():
      _load_table("res://data/items/weapons.json", weapons)
      _load_table("res://data/abilities/fighter.json", abilities)
      _load_table("res://data/enemies/bandits.json", enemies)

  func _load_table(path: String, target: Dictionary) -> void:
      var file = FileAccess.open(path, FileAccess.READ)
      var json = JSON.parse_string(file.get_as_text())
      for row in json.rows:
          target[row.id] = row

  func get_weapon(id: String) -> Dictionary:
      return weapons.get(id, {})

  func get_all_weapons() -> Array:
      return weapons.values()

  func get_weapons_by_rarity(rarity: String) -> Array:
      return weapons.values().filter(func(w): return w.rarity == rarity)

  Usage in Game

  # In combat or inventory code:
  var sword = GameData.get_weapon("w_002")
  print(sword.name)       # "Steel Sword"

  print(sword.value)      # 150
